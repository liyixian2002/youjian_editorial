00 · BACKGROUND

为什么做这个项目？

小投入、小组织，搞清楚 AI 时代内容营销的新生产力组织方式。

📌 项目起源
我们想通过小投入、小组织，搞一个 Agent Team，实现内容和热点营销的价值产出—— 1 分钟出判断，5 分钟出 Sample，8 分钟出 Brief。 一方面是做一个方向探索，一方面也是在过程中找到 AI 项目的新的生产力组织方式。
⏰ 投入方式
每人每天大约 2–3 小时，利用业余时间跟着项目起步。有运营负责训练 Agent，有产品负责门户和架构设计，有技术负责记忆系统搞建、harness 工程和底座对接。
🚀 一周上线
四个人，一周时间，有间编辑部就跟着跑起来了。 第一个闭环已经跑通：热点输入 → 1 分钟判断 → 5 分钟 Sample → 8 分钟 Brief。

01 · CONTEXT

我们要解决什么问题？

内容营销不缺 idea，缺的是快速判断、改造、落地 idea 的机制。

灵感来了，判断慢
负责人看到热点或竞品动作，问"这个我们能不能跟"——但团队需要调研、讨论、对齐，往往错过窗口期。
方向对了，落地难
即使判断方向可行，还要改造 idea、拆方案、写脚本、找素材、做 demo，每一步都是人力消耗。
核心矛盾
负责人脑子里的灵感，和团队可以判断、讨论、执行、复盘的内容方案之间，
缺少一套 快速转化机制。
02 · SOLUTION

过去靠人，现在靠谁？

从被动响应到主动观察，从人力拼接到 Agent 协作。

无调度涌现，有调度交付。

🐢  过去的做法
热点判断
人工搜索、开会讨论、对齐方向，1-2天
方案生成

内容产出
文案、脚本、素材分头找人，排期协调
风险把控
依赖个人经验，事后发现问题

热点判断
主编 Agent + 黑脸 Agent，1 分钟给出判断、风险和建议
方案生成
营销策略 + Demo 执行 Agent，5 分钟出 Sample，8 分钟出可评审 Brief
内容产出
内容选题 Agent 统一输出脚本、图文、达人 brief
风险把控
舆论风险 Agent 事前扫描，黑脸 Agent 主动反驳

05 · LESSONS

过程中碰到的问题

哪些解决了，哪些还没解决。

■ 已解决
✓
Sample 生成成功率低
第二场会议的交付物 Sample 有概率不显示，可能是生成失败，也可能是生成成功写入表中但未读取展示。解法：更换生成成功率更高的模型。
✓
Sample 素材与议题不符
设计师 Agent 出的 Sample 素材跑偏，例如到餐的策略出的是外卖的文案。解法：多喂给 Agent 外卖场景以外的素材和词库，扩充训练覆盖面。
✓
Agent 输出污染
多 Agent 工作流最终输出夹带"skill xxx is loading"、"正在调用 xx skill"等过程信息，直接透出给用户影响体验。解法：在 SSE 事件解析函数里增加对 thinking 和 tool 类型帧的判断，遇到这两种类型直接丢弃不拼接，只保留真正的回答内容输出给用户。
■ 未解决
△
切换 Session 后 Sample 生成中断
切换议题开启新 Session 后，Sample 生成任务会中断，成功率为 0。该问题目前依旧未解决，需要进一步排查 Session 切换时的任务生命周期管理。
△
前端交互常规
B 端 PM 做 C 端交互存在短板，门户体验偏工具化，缺乏消费级产品的交互质感。暂无解法。
△
三阶段交付物生成超时
三阶段的交付物无法在规定时间内生成完成。解法：更换速度更快的模型，但生成质量无法保证，速度与质量之间存在取舍。
△
行为数据缺失，无法做漏斗分析
想做更具体的数据分析，但后台数据不支持。例如：用户召开会议后有没有看判断卡？多少人继续产出了 Sample？多少人出了 Brief？都在哪个环节退出？按原来模式埋点需要 5pd+。暂无解法。
△
Brief 跳转 Wiki 成功率低
生成的完整 Brief 跳转到 Wiki 成功率不高。暂无解法。
